# Client-Facing Language Policy

## Permanent Global Rule

The platform must never expose AI terminology in the user interface or client communications.

### Prohibited Terms (Client-Facing)

Do not display any of these in client-facing screens:

- AI
- Artificial Intelligence
- GPT / Claude / Gemini / LLM
- OpenAI / Anthropic
- Prompt / Model / Temperature / Tokens / Context Window
- Reasoning Engine / AI Score / AI Quality / AI Confidence
- AI Persona / AI Assistant / AI Agent / AI Tool / AI Platform
- AI Engine / AI System / AI Powered / AI-Generated
- Generated by AI / Generated by Claude

### Scope

This applies across all client-facing surfaces:

- UI labels, headings, buttons, navigation
- Emails, notifications, toasts
- Dashboards, reports, analytics
- Help text, onboarding, documentation
- PDF exports, presentations
- Error messages, status indicators
- Audit logs (client-facing)
- Empty states, loading states
- Tooltips, placeholders

### Exceptions

AI terminology may appear only in:
- Developer/admin diagnostic tools (e.g., `ai-test.tsx`)
- Internal code comments
- Backend API documentation
- Admin-only logs and dashboards

### White-Label Principle

The platform must feel like a premium communications operating system, not an AI tool. Clients should believe they are using an intelligent communications platform that helps them make better decisions. How it achieves those outcomes is an internal implementation detail.

### Compliance

All code changes must be reviewed against this policy before merge. The `ai-test.tsx` developer tool and the `Event Perfekt Agent` branding are the only permitted uses of AI-related language.
