Bug 1 — Unicode escape sequences rendering as raw text instead of characters
The card displays show \u23f0 and \u2014 as visible text instead of rendering as the actual characters ⏰ and —. This means the data was inserted as escaped Unicode strings rather than as decoded characters.
Fix: Search the partner_directory table (and any related rows) for the literal strings \u23f0, \u2014, \u2026, \u2018, \u2019, \u201c, \u201d, \u2013 and replace each with the actual decoded character:

\u23f0 → ⏰ (or just remove the emoji and write "Next action:" in plain text — cleaner)
\u2014 → — (em dash)
\u2013 → – (en dash)
\u2026 → … (ellipsis)
\u2018 \u2019 → ' ' (smart quotes)
\u201c \u201d → " " (smart double quotes)

Run this as a database-side fix on every text field in the partner_directory table. Use UTF-8 encoding for the table going forward.
Recommended: Remove the ⏰ emoji entirely from the Next action display. Card UIs read cleaner with plain "Next action: 18/06/2026 — Send Template 1 cold outreach re pro bono training" than with an emoji that may not render consistently across browsers and devices.
