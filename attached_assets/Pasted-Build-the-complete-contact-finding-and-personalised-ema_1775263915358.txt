Build the complete contact finding and personalised email drafting system. When the daily agent finds a company, it must automatically find the right person to contact and draft a personalised email from the correct sender.

# PART 1 — CONTACT FINDING ENGINE

Create a function: `findDecisionMaker(companyName, companyWebsite, country, companyNumber)`

This runs automatically for every prospect the daily agent adds AND is available manually on the Contact Intel tab.

## TARGET TITLES (in priority order)

Search for people with these titles, in this exact priority order:

1. Events Director / Director of Events
2. Marketing Director / Director of Marketing
3. Head of Events
4. Head of Marketing
5. Events Manager
6. Marketing Manager
7. Brand Director / Head of Brand
8. Sales Director / Head of Sales / Commercial Director
9. Chief Marketing Officer / CMO
10. Managing Director / MD
11. Chief Executive Officer / CEO
12. Founder (if small company)

Always prefer Events or Marketing titles first. Only go to MD/CEO/Founder if no Events or Marketing contact found.

## SOURCE WATERFALL (try each in order, stop when result found)

### Source 1: RocketReach API
API key already in Replit env vars. Check env for ROCKETREACH_API_KEY.
```
GET https://api.rocketreach.co/api/v2/lookupProfile
params: { name: '[target name]', current_employer: '[company]' }

OR search by company:
GET https://api.rocketreach.co/api/v2/searchProfile  
params: { current_employer: '[company]', title: '[title keyword]' }
```
Return: name, title, email, LinkedIn URL, confidence score
Mark source as: "RocketReach" confidence = High if SMTP verified

### Source 2: Hunter.io API
API key in env as HUNTER_API_KEY.
```
GET https://api.hunter.io/v2/domain-search
params: { domain: '[company domain]', api_key: '[key]', type: 'personal' }
```
Filter results by title priority list above
Return: name, title, email, LinkedIn (if available)
Mark source as: "Hunter.io" confidence = Medium (not always SMTP verified)

### Source 3: Companies House (UK only)
For UK companies, use company number from Companies House API:
```
GET https://api.company-information.service.gov.uk/company/[number]/officers
```
Returns: director names and appointment dates
For each director found, try to find email via Hunter.io domain search
Mark source as: "Companies House" confidence = High (verified director)
Note: Companies House gives name and title but rarely email — combine with Hunter.io for email

### Source 4: Website Scraping
Scrape the company's /team, /about, /about-us, /people, /leadership pages
```
const pagesToTry = [
  '/team', '/about', '/about-us', '/people', 
  '/leadership', '/our-team', '/staff', '/who-we-are'
]
```
For each page: extract names and job titles using pattern matching
Look for title keywords from priority list
Then try Hunter.io to find email for matched name at company domain
Mark source as: "Website" confidence = Medium

### Source 5: Agent Research (fallback)
If all above fail, call callAgent() with task: 'contact_intel'
Prompt: "Find the most senior person responsible for events, marketing, or brand at [company name] in [country]. Search their website, LinkedIn, and public sources. Return: full name, job title, email address (if findable), LinkedIn URL, and confidence level (High/Medium/Low). Only return real people with verifiable roles."
Mark source as: "Agent Research" confidence = Low/Medium based on Agent response

## RESULT FORMAT

Return the best contact found as:
```typescript
{
  name: string,
  firstName: string,
  lastName: string, 
  title: string,
  email: string | null,
  emailPattern: string | null, // e.g. firstname.lastname@company.com
  linkedIn: string | null,
  confidence: 'High' | 'Medium' | 'Low',
  source: string,
  sourceNote: string, // e.g. "SMTP-verified via RocketReach"
  alternativeContacts: [ // up to 3 alternatives
    { name, title, email, source }
  ]
}
```

Display exactly like the Mi Homes example in the screenshot:
- EP-VERIFIED CONTACT header (green)
- Name bold, Title alongside
- Email as clickable link
- Email pattern shown
- LinkedIn Profile link
- Confidence badge (HIGH/MEDIUM/LOW)
- Source note in small text
- Warning if from public records: "Found in public records — treat as reliable, but always confirm before sending"
- Alternative contacts listed below

## AUTO-RUN ON DAILY PROSPECTS

When the daily prospecting agent adds a new prospect:
- Automatically call findDecisionMaker() for each new prospect
- Store result in prospect record
- Show EP-VERIFIED CONTACT card on the prospect card in Pipeline
- If no contact found: show "Contact not found — try Contact Intel tab"
- Do not block prospect from being added if contact not found

# PART 4 — COMPLETE DAILY FLOW END TO END

The complete automated daily flow must work like this:

1. 7am: Daily agent finds companies with triggers
2. For each company: findDecisionMaker() runs automatically
3. Contact stored against prospect record
4. Agent drafts personalised email using correct template + correct sender
5. Email saved as pending_approval against the prospect
6. 7:30am: Morning briefing sent to all managers showing new prospects + contacts found + emails ready for review
7. Staff opens Outreach tab, sees drafted emails with contact name, subject line preview
8. Reviews each email, edits if needed, clicks Approve
9. Email sends from correct sender address via SendGrid
10. Activity logged: sent by [sender name] to [contact name] at [email] on [date]
11. Sequence continues automatically for remaining touches
