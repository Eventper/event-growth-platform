## Why fixtures

The six fixes are reported as implemented and the build is healthy — but a healthy server proves the code *runs*, not that the gates *behave*. With `passedGate=0`, none of the new logic was exercised on real data. So: load a small set of **saved fixture tenders** that deliberately hit every tricky case, run the sweep over them, and prove the behaviour. This closes the fixes for real instead of on a build signal.

---

## Part A — Coverage additions (config)

Add these to the discovery config **before** running the fixtures, so the fixtures test them too.

**1. University & alumni EVENTS** → add to the Events lane keywords:
`graduation, congregation, degree ceremony, open day, alumni event, alumni reunion, alumni engagement, university conference, freshers fair, awards ceremony, gala dinner, fundraising gala, donor event, commemoration event`

> **Guard — "university" is NOT a qualifier on its own.** Universities buy huge amounts of non-event work (the Northumbria T25-0047 business-consultancy lots proved this). These terms qualify because they are *event* terms — keep the lane-discipline rule: a university tender qualifies only when an **event/ceremony** term is present, never on "university"/"college" alone.

**2. Event decoration & theming** → add to the Experiential/Creative lane keywords, **GATED**:
`event decoration, event theming, event styling, venue dressing, venue styling, set dressing, stage décor, draping, backdrop design, marquee dressing, prop hire, themed event, event décor, floral styling (events)`

> **Guard — décor is a false-friend with the construction trade.** "Painting and decorating", "decorator", "interior decoration", "redecoration" are **building works** and must stay **EXCLUDED**. So décor/theming terms qualify **only when an event/venue/ceremony/gala term co-occurs** (reuse the F4 DELIVERY_KEYWORDS co-occurrence check). Décor word alone → do not qualify; if it matches a construction-exclude term → drop.
