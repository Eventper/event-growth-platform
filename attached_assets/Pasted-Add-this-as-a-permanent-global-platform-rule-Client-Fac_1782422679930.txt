## Client-Facing Language Policy

The platform must never expose AI terminology in the user interface or client communications.

Do not display:

* AI
* Artificial Intelligence
* GPT
* Claude
* Gemini
* LLM
* OpenAI
* Anthropic
* Prompt
* Model
* Temperature
* Tokens
* Context Window
* Reasoning Engine
* AI Score

These are implementation details and should never appear in client-facing screens.

Instead use professional business language.

Examples:

Instead of:
"AI is generating your email..."

Use:
"Preparing your communication..."

Instead of:
"Claude selected the PR persona."

Use:
"Your communication has been prepared using the PR communications framework."

Instead of:
"AI Quality Score"

Use:
"Communication Quality"

Instead of:
"AI Confidence"

Use:
"Quality Assessment"

Instead of:
"Model"

Use:
"Writing Style"

Instead of:
"Prompt"

Use:
"Communication Brief"

Instead of:
"AI Persona"

Use:
"Communication Strategy"

Instead of:
"Reasoning"

Use:
"Editorial Review"

Instead of:
"Generated by Claude"

Use:
"Prepared"

## Internal vs Client Views

Create two completely separate experiences.

### Internal (Admin / Developer)

Can display:

* Model
* Provider
* Prompt
* Token usage
* Latency
* Cost
* Debug logs
* Retry attempts

### Client Workspace

Must only display business language:

* Communication
* Draft
* Strategy
* Editorial Review
* Writing Style
* Quality Assessment
* Brand Voice
* Ready for Review
* Ready to Send

Never expose technical implementation details.

---

## White-Label Principle

The platform must feel like a premium communications operating system, not an AI tool.

Clients should believe they are using an intelligent communications platform that helps them make better decisions.

How it achieves those outcomes is an internal implementation detail.

This rule applies across the entire SaaS, including:

* UI
* Emails
* Reports
* Dashboards
* Notifications
* Audit logs
* Help text
* Onboarding
* Documentation (client-facing)
* PDF exports
* Presentations

The only place AI terminology may appear is in developer/admin diagnostics, never in the client experience.
