Build a premium Décor Service, Venue Scanner, Guest Management and Event Design Studio inside our app. This should be one connected system, not separate disconnected tools.

The goal is to create a feature that allows us to plan an event visually and operationally from one place. It must combine:

décor planning

venue scanning

guest management

seating logic

2D layout generation

3D visualisation

prompt-based automation

walkthrough/presentation outputs

This should feel like a professional event planning and design platform, similar in ambition to Homestyler and 3D Event Designer, but built specifically for event décor, seating, venue planning and guest flow.

Core product vision

We want a planner to be able to:

create or open an event

enter event details

enable Guest Management

enable Venue Scanner

have the system automatically generate the venue shell in 2D and 3D

have the system use guest numbers, seating rules, zones and décor preferences to automatically generate a draft layout

manually edit the design if needed

switch from 2D to 3D instantly

preview the event visually and operationally

use the final result for client presentation and internal delivery planning

This is not just a seating chart tool.
This is not just a generic room designer.
This must be a connected event décor and guest planning design studio.

Main principle

The system should work like this:

Event Record + Venue Scanner + Guest Management + Décor Library + Prompt Input + Layout Rules = automatic 2D and 3D event plan

That is the key requirement.

Automatic layout generation from guest data

This is one of the most important features.

When Guest Management is enabled, the system should automatically generate a floor plan based on:

guest count

VIP count

seating type

tables per guest count

venue size

event type

selected décor items

required zones

Examples:

180 guests with tables of 10 should suggest 18 tables, then adjust for VIP or special tables

a conference should generate rows or classroom style, not banquet unless selected

if registration is enabled, registration desk should appear near the entrance

if VIPs exist, create a premium zone

if the hall is too small, show a capacity warning

This logic must be based on event rules, not random placement.

Automation flow required

The automation flow should work like this:

Step 1
User creates or opens an event

Step 2
User enters or imports event details

Step 3
User enables Guest Management

Step 4
User enables Venue Scanner or uploads venue/floor plan

Step 5
System generates editable 2D venue shell and 3D venue shell

Step 6
System reads guest numbers, seating requirements and prompt if provided

Step 7
System generates a first-pass event layout automatically

Step 8
User edits furniture, colours, materials, zones and styling

Step 9
User switches to 3D preview

Step 10
User saves, presents or exports the design
